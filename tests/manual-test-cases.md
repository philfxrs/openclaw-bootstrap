# 手动测试用例

本文档列出 OpenClaw Bootstrap Installer 的手动测试场景。

## 测试环境准备

- 准备一台干净的系统（或虚拟机/容器）
- 确认系统满足最低要求
- 确认网络可用

---

## 测试用例

### TC-01: 全新 macOS 安装

**前提**: macOS 系统，已安装 Node.js >= 22，未安装 OpenClaw

**步骤**:
1. 克隆仓库
2. 运行 `chmod +x bootstrap/macos/install-openclaw-macos.sh`
3. 运行 `./bootstrap/macos/install-openclaw-macos.sh --install`
4. 按提示完成安装

**预期结果**:
- 预检全部通过
- 安装成功完成
- 配置向导正常运行
- 验证通过
- 日志文件已创建

---

### TC-02: 已安装旧版本再执行安装

**前提**: 已安装旧版 OpenClaw

**步骤**:
1. 运行安装脚本
2. 观察是否检测到已安装版本
3. 选择相应操作

**预期结果**:
- 正确检测到已安装版本
- 提示选择：跳过 / 升级 / 修复 / 重置配置
- 不会粗暴覆盖已有安装

---

### TC-03: Node.js 缺失

**前提**: 系统未安装 Node.js

**步骤**:
1. 确认 `node --version` 报错
2. 运行安装脚本

**预期结果**:
- 预检报告 Node.js 为 FAIL
- 输出结构化错误信息
- 给出安装建议和命令
- 安装流程中止

---

### TC-04: Node.js 版本过低

**前提**: Node.js 版本 < 22.0.0

**步骤**:
1. 确认 `node --version` 低于要求
2. 运行安装脚本

**预期结果**:
- 预检报告 Node.js 版本过低
- 输出升级建议
- 安装流程中止

---

### TC-05: 网络不可用

**前提**: 断开网络或阻止访问 npmjs.org

**步骤**:
1. 断开网络
2. 运行安装脚本

**预期结果**:
- 预检报告网络连通性 FAIL
- 输出网络排查建议
- 安装流程中止

---

### TC-06: PATH 未生效

**前提**: Node.js 安装了但 npm 全局目录不在 PATH 中

**步骤**:
1. 安装 OpenClaw
2. 确认 `openclaw` 命令不可用
3. 运行 `--repair`

**预期结果**:
- 修复模块检测到 PATH 问题
- 提供修复选项
- 修复后 PATH 正确

---

### TC-07: 升级

**前提**: 已安装 OpenClaw

**步骤**:
1. 运行 `--upgrade`

**预期结果**:
- 检测当前版本
- 备份配置
- 执行升级
- 升级后验证

---

### TC-08: 升级失败

**前提**: 模拟升级失败（如断网）

**步骤**:
1. 安装 OpenClaw
2. 断开网络
3. 运行 `--upgrade`

**预期结果**:
- 升级失败
- 输出回滚指引
- 不会损坏现有配置

---

### TC-09: 修复

**前提**: 安装存在问题

**步骤**:
1. 运行 `--repair`

**预期结果**:
- 逐项诊断
- 发现并尝试修复问题
- 输出修复结果

---

### TC-10: 重置配置

**前提**: 已安装并配置

**步骤**:
1. 运行 `--reset-config`
2. 确认备份

**预期结果**:
- 备份现有配置
- 重置为默认配置
- 输出备份位置

---

### TC-11: 非交互模式安装

**前提**: 全新环境

**步骤**:
1. 运行 `--install --non-interactive`（macOS）
2. 或 `-Install -NonInteractive`（Windows）

**预期结果**:
- 不出现任何交互提示
- 使用默认值完成所有操作
- 跳过项记录到日志

---

### TC-12: 验证

**前提**: 已安装 OpenClaw

**步骤**:
1. 运行 `--verify`

**预期结果**:
- 输出各项验证结果
- 全部通过或指出具体问题

---

### TC-13: 白名单校验

**步骤**:
1. 修改白名单文件，移除 npmjs.org
2. 运行安装

**预期结果**:
- 安装被阻止
- 输出域名不在白名单的错误

---

### TC-14: Verbose 模式

**步骤**:
1. 运行 `--install --verbose`

**预期结果**:
- 输出详细调试信息
- 包含 `[DEBUG]` 前缀的额外日志

---

### TC-15: 帮助信息

**步骤**:
1. 运行 `--help`

**预期结果**:
- 显示完整帮助信息
- 列出所有参数和示例
